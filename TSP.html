Section 02

About the Problem

One of the most studied combinatorial optimisation problems in computer science.

The Travelling Salesman Problem (TSP) asks: given a set of cities and the costs to travel between each pair, what is the shortest possible route that visits every city exactly once and returns to the start?

TSP is NP-hard. With n cities there are (n−1)!/2 distinct tours. With 10 cities there are 181,440; with 20 there are over 60 quadrillion. Brute-force is infeasible.

Dynamic Programming (Held–Karp) solves TSP exactly in O(n² · 2ⁿ) time by storing optimal sub-paths to each subset of cities. Branch and Bound prunes infeasible subtrees using a lower bound computed from the reduced cost matrix — often much faster in practice.

DP Subproblem

dp[S][v] = minimum cost to reach city v having visited exactly the cities in set S.

Bitmask DP

Sets are represented as bitmasks. There are 2ⁿ subsets × n cities = n · 2ⁿ states.

B&B Bounding

Lower bound = current cost + reduced matrix cost. Prune if lb ≥ best known tour.

Pruning Power

Effective pruning makes B&B far faster than brute force. Worst case is still O(n!).

Held–Karp DP — Step by Step
  1. Start at city 0. Initialize dp[{0}][0] = 0; all others = ∞.
  2. For each subset S containing city 0, for each city u in S, for each city v not in S: relax dp[S∪{v}][v] = min(dp[S∪{v}][v], dp[S][u] + cost[u][v]).
  3. After filling all states, find the optimal tour cost: min over u ≠ 0 of dp[all cities][u] + cost[u][0].
  4. Trace back through parent pointers to reconstruct the optimal path.
Complexity
O(n² · 2ⁿ)

Exponential in n but far better than factorial brute force. Practical up to ~20 cities.

Branch & Bound — Step by Step
  1. Compute the reduced cost matrix. The reduction cost is the initial lower bound lb₀.
  2. Branch: create child nodes for each unvisited city. Each child extends the partial path by one city.
  3. For each child, update the matrix (set row/col to ∞), reduce again. New lb = parent lb + edge cost + new reduction.
  4. Priority queue ordered by lb (Best-First). Expand the node with the smallest lb.
  5. When all cities are visited, record the complete tour as the current best upper bound.
  6. Prune any node with lb ≥ best. Empty queue → optimal tour found.
Worst-case Complexity
O(n! / pruning)

Worst case is factorial, but aggressive pruning makes it vastly faster in practice.
